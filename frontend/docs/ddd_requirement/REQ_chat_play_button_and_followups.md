# REQ: Chat "Press to Play" Button and Follow-up Questions

## Summary

After every assistant reply in the Chat sidebar, a "Press to play" button and three static follow-up question chips are injected below the message bubble. Clicking the button applies a fixed filter preset to the US athlete map and navigates the view to the Map Page. Clicking a question chip immediately sends that question as a new user message.

---

## Actors

- **User** — interacts with the Chat sidebar and the map.
- **AI Agent** — produces assistant messages in the Chat.

---

## Primary Flows

### 1 — Viewing the button and chips after an assistant reply

1. The AI agent returns a reply; it is rendered as an assistant message bubble.
2. Immediately below the bubble (left-aligned, same column), the "Press to play" button appears.
3. Below the button, three follow-up question chips appear in a vertical stack.

### 2 — Pressing "Press to play"

1. The user clicks the button.
2. The map filter state is overwritten with the preset (see **Filter Preset** below).
3. If the Content Page slide is active, the view transitions back to the Map Page.
4. The button remains fully active; pressing it again re-applies the same preset.

### 3 — Clicking a follow-up question chip

1. The user clicks one of the three question chips.
2. The question text is sent immediately as a user message (same as typing and pressing Send).
3. The chat continues normally from that point.

---

## Filter Preset (applied on button press)

| Filter | Value |
|---|---|
| State | CA (California) |
| Game | Both (Olympian and Paralympian selected) |
| Season | Both (Summer and Winter selected) |
| Medal | Gold only (silver, bronze, and no-medal are deselected) |
| Sport | All disciplines (no sport restriction) |
| Athlete | No selection (filter skipped — all athletes eligible) |
| City | Los Angeles |

---

## Follow-up Question Texts (static placeholders)

1. "Who won gold in LA 1984?"
2. "Which sport had the most CA athletes?"
3. "Show me Paralympic winners from California"

These are static strings — the same three questions appear after every assistant reply regardless of context.

---

## UI Layout

```
[ assistant message bubble                        ]

[ ▶ Press to play                                ]

[ Who won gold in LA 1984?                       ]
[ Which sport had the most CA athletes?          ]
[ Show me Paralympic winners from California     ]
```

The button and chips are visually attached to the assistant turn — they do not appear after user messages or system messages.

---

## Acceptance Criteria

- AC-1: The "Press to play" button is present below every assistant message bubble (not user or system messages).
- AC-2: The three follow-up chips are present below the button, after every assistant message.
- AC-3: Clicking the button applies the exact filter preset defined above.
- AC-4: After the button is clicked, if the Content Page is showing, the view slides back to the Map Page.
- AC-5: After the button is clicked, it remains active and can be pressed again to re-apply the preset.
- AC-6: Clicking a question chip immediately sends it as a user message to the agent.
- AC-7: The three question texts are the static strings listed above (exact text may be adjusted in design/implementation, but must be semantically equivalent).
- AC-8: Filter state changes triggered by the button are reflected on the map in real time (same behavior as manual filter changes).

---

## State Sharing

The filter state (state, game, season, medal, sport, athlete, city) currently lives inside `MapWithFilter`. Chat is a sibling component. Both are children of `ResizableLayout`. To allow Chat to trigger filter changes, the filter state and the map-navigation state (Map vs Content page) must be lifted to the nearest common ancestor. No global state library is required.

---

## Errors and Edge Cases

- **Chat unavailable**: The button and chips must not appear on system error messages or during typing-indicator bubbles. They appear only on fully rendered assistant message bubbles.
- **No athletes match preset**: The map may render with no dots; this is acceptable — the filter logic already handles empty results.
- **Los Angeles not in city list**: If no city with name "Los Angeles" exists in the dataset, the city filter is effectively empty (no city restriction). This should be handled gracefully without error.

---

## Out of Scope

- Dynamic follow-up questions generated by the AI (questions are static placeholders only).
- Saving or persisting the preset filter state across sessions.
- Multiple different presets or per-message presets.
- The button triggering any AI call itself — it only changes filter state and navigation.
- Analytics or tracking of button presses.
- Accessibility (keyboard navigation, ARIA) beyond what Tailwind/browser defaults provide.
